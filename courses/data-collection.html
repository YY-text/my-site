数据采集与处理

商务数据分析与应用专业核心课程

课程介绍

数据采集与处理课程是商务数据分析与应用专业的重要课程，主要介绍如何从各种来源采集数据，并进行预处理和清洗。通过本课程的学习，学生将掌握数据采集的方法和工具，以及数据预处理的技术，为后续的数据分析做准备。

课程目标

1. 掌握数据采集的基本方法和工具
2. 学会使用网络爬虫技术获取数据
3. 掌握数据清洗和预处理技术
4. 了解数据集成和存储方法
5. 能够处理实际中的数据问题

第1章 数据采集概述

1.1 数据来源

通俗讲解

数据来源就像我们收集信息的渠道，有的来自网络，有的来自数据库，有的来自Excel表格。就像做饭一样，不同的食材需要从不同的地方采购。

核心知识点
  • 内部数据：企业内部系统产生的数据
  • 外部数据：政府、第三方机构、公开数据
  • 网络数据：网站、社交媒体、电商平台
  • 传感器数据：物联网设备产生的数据
示例：常见数据来源 
数据来源分类：

1. 公开数据平台：
   - 国家统计局
   - 世界银行开放数据
   - Kaggle数据集

2. 网络数据：
   - 电商网站商品信息
   - 社交媒体评论
   - 新闻网站文章

3. 企业内部：
   - 销售记录
   - 客户信息
   - 财务报表

1. 以下哪个不是数据来源？

2. 企业销售记录属于？

3. Kaggle数据集属于？

4. 社交媒体评论属于？

5. 选择数据来源时应优先考虑？

6. 以下哪个属于传感器数据？

7. 政府公开数据的特点是？

8. 数据采集的第一步是？

9. 内部数据相比外部数据的优势是？

10. 关于数据来源，正确的说法是？

1.2 数据类型

通俗讲解

数据类型就像食材分类，有的是蔬菜，有的是肉类，有的是调料。结构化数据像超市货架上整齐的商品，非结构化数据像菜市场上摆放的各种食材。

核心知识点
  • 结构化数据：表格形式，如数据库、Excel
  • 非结构化数据：文本、图片、音频、视频
  • 半结构化数据：JSON、XML等格式
  • 不同类型需要不同的处理方法

1.3 采集原则

通俗讲解

数据采集就像买菜，要新鲜、适量、符合需求。不能买太多吃不完，也不能买太少不够用，还要注意食材的质量。

核心知识点
  • 合法性：遵守法律法规和网站规则
  • 必要性：只采集需要的数据
  • 时效性：及时更新数据
  • 完整性：保证数据全面完整

1.4 质量要求

通俗讲解

数据质量就像食材的新鲜度，新鲜的食材做出的菜才好吃。好的数据才能得出可靠的分析结果。

核心知识点
  • 准确性：数据真实可靠
  • 完整性：没有缺失值
  • 一致性：数据格式统一
  • 及时性：数据及时更新

第2章 网络爬虫基础

2.1 HTTP协议

HTTP是浏览器和服务器通信的协议，就像邮递员送信的规则。

2.2 HTML基础

HTML是网页的骨架，用标签组织内容，爬虫需要解析这些标签。

2.3 爬虫原理

爬虫模拟浏览器访问网页，解析内容，提取需要的数据。

2.4 反爬虫策略

网站会通过验证码、User-Agent检测等方式防止爬虫，需要合理应对。

第3章 爬虫工具与库

3.1 Requests库

Requests是Python最常用的HTTP请求库，简单易用。

3.2 Beautiful Soup

Beautiful Soup用于解析HTML和XML文档，方便提取数据。

3.3 Scrapy框架

Scrapy是强大的爬虫框架，适合大规模数据采集。

3.4 Selenium自动化

Selenium可以模拟真实浏览器操作，处理动态网页。

第4章 API数据采集

4.1 API概述

API是应用程序接口，提供标准化的数据访问方式。

4.2 RESTful API

RESTful是一种API设计风格，使用HTTP方法操作资源。

4.3 认证方式

常见的API认证方式有API Key、OAuth等。

4.4 JSON解析

JSON是API常用的数据格式，需要学会解析和处理。

5.2 Excel文件

5.3 JSON文件

JSON文件存储结构化数据，格式灵活。

5.4 XML文件

XML是可扩展标记语言，适合存储层次化数据。
